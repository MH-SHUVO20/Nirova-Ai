# Nirova-Ai Azure Infrastructure

This folder contains production-ready Bicep templates for deploying the Nirova-Ai platform on Azure.

## Architecture
- **FastAPI backend** and **React frontend** (containerized, deployed to Azure Web Apps for Containers)
- **Cosmos DB (MongoDB API)** for data
- **Azure Cache for Redis**
- **Azure Storage Account** (Blob)
- **Azure Key Vault** (secrets, managed identity)
- **Azure Application Insights** (monitoring)
- **Azure Container Registry** (for images)
- **Private Endpoints** for databases and storage

## Deployment
1. Review and update parameter files in `parameters/` for your environment.
2. Update `tenantId` and `objectIds` for Key Vault access.
3. Deploy using Azure CLI:
   ```sh
   az deployment group create --resource-group <your-rg> --template-file main.bicep --parameters @parameters/dev.parameters.json
   ```
4. For production, use `prod.parameters.json`.

## Security & Best Practices
- All secrets and connection strings are stored in Key Vault.
- Managed identities are used for secure access.
- Private endpoints restrict public access to databases and storage.
- Application Insights is enabled for monitoring.
- All resources are tagged and parameterized.

## Customization
- Edit module files in `modules/` to adjust resource settings.
- Add more modules as needed for networking, VNETs, etc.
